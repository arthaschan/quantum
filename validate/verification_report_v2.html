⚠️ 关键发现：Spearman ρ 相关性 Bug

Bug 位置

scripts/all5_experiments.py 第 175-182 行 和 scripts/e2_subgroup.py 第 58-66 行

Bug 代码

def spearman_r(x, y):
    n = len(x)
    rx = {v: i+1 for i, v in enumerate(sorted(set(x)))}
    ry = {v: i+1 for i, v in enumerate(sorted(set(y)))}
    rx_arr = [rx[xi] for xi in x]
    ry_arr = [ry[yi] for yi in y]
    d2 = sum((a-b)**2 for a,b in zip(rx_arr, ry_arr))
    return 1 - 6*d2/(n*(n**2-1))

Bug 分析

  • 公式 ρ = 1 - 6Σd²/(n(n²-1)) 假设两个变量的秩都是 1,2,...,n 且 无 ties
  • unique_pf 只有 4 个不同值 (1,2,3,4)，168 个数据点中存在大量 ties
  • 68 个点 unique_pf=2 → 全部被分配相同的秩 2
  • 82 个点 unique_pf=3 → 全部被分配相同的秩 3
  • 公式假设 n 个不同的秩，但实际 x-秩被压缩到 4 个值，导致 ρ 被严重放大

结论：d=2039 的 ratio=0.327 是可复现的（相同种子），但它是单一随机态的极端离群值。 多态中位数 ratio≈0.80（约 20% 优势），远不如论文声明的 67.3% 极端。 原始实验的 "1 state" 设计导致采样方差极大。

对论文修改的建议

  1. 【紧急】修复 Spearman ρ 计算。 将 scripts/all5_experiments.py 和 scripts/e2_subgroup.py 中的手动 Spearman 实现替换为 scipy.stats.spearmanr。修正后 ρ ≈ 0.03 (p ≈ 0.7)，需要重写 §5.3.1、§7.1 和 G1 部分的全部结论。
  2. 【重要】多态平均。 当前大素数实验 (E3) 每个维度仅使用 1 个随机态， 导致单点 ratio 波动极大（d=2039 波动范围 0.33-1.50）。 建议每个维度至少使用 10+ 个随机态，报告中位数和置信区间。
  3. 修正 G1/G2/G3 的解释框架。 由于 p-1 光滑度与 ratio 不存在显著相关， "群论框架"作为协议性能的代数解释需要重新表述。实验现象（原根敏感性、子集挽救）本身仍然有效， 但需要寻找新的理论解释。
  4. 补充与 MLE（最大似然估计）基线的对比实验（d≤30），这是当前最明显的缺失基线。
  5. 论文的 E1/E3/E7/W1 核心数值实验代码正确，数据可靠，可作为投稿基础。
